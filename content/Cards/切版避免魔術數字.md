切版時，固定像素寬度常常是魔術數字——看起來是任意數值，實際上是由容器尺寸反推計算出來的結果。這類寫法在任何 CSS 框架或純 CSS 中都應該避免。

## 案例

以 Tailwind 為例：

```
外層容器 max-w-[1800px]
  └─ section p-6（padding 24px × 2 = 48px）
       └─ 可用寬 = 1800 - 48 = 1752px
            └─ flex gap-6（gap 24px）
                 └─ 左右各得 = (1752 - 24) ÷ 2 = 864px
```

`864px` 實際上就是在 1920px 螢幕下的一半空間，改成 `50%` 或 `flex: 1` 語意更清楚，也不依賴特定螢幕寬度。

## 原則

- 固定 px 寬度如果只是計算比例的結果，應改用 `50%`、`flex: 1`、`calc()` 等語意化寫法
- Label、表格欄、UI 控件等以內容為基準的元素，保留固定 px 是合理的
- 魔術數字難以維護，日後改容器尺寸時容易忘記同步更新
