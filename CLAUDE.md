# Obsidian Memory Vault

## Vault 結構

組織方式：

- `Inbox/` — 所有新內容（包含日記）先進這裡，之後再整理
- `Atlas/` — MOC（Map of Content）索引筆記，`Home.md` 是整個 vault 的入口
- `Cards/` — 單一概念的原子筆記
- `Projects/` — 以主題為單位的資料夾，將同類的 Cards 放在一起

**日記存放路徑**：`Inbox/YYYY-MM-DD.md`

## 組織原則

- 每張筆記只記錄一個知識點（原子筆記）
- `Projects/` 資料夾代表主題，不代表有開始或結束的專案
- `Atlas/` 的 MOC 負責跨主題索引，`Projects/` 資料夾只收同主題的筆記
- 搜尋知識點：透過 `Ctrl+O`、`Ctrl+Shift+F`、MOC 連結或標籤

操作原則：先捕捉、後整理，降低記錄阻力。
